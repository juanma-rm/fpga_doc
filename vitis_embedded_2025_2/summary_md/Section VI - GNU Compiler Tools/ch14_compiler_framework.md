# Chapter 14 — Compiler Framework

This chapter discusses the common features of the MicroBlaze/MicroBlaze-V, and Cortex A9, A53, A72, and R5 processor compilers.

## GNU Tool Flow

The GNU compiler is a wrapper that calls four executables sequentially:

```
Input C/C++ Files
  → cpp0 (Pre-processor)
  → cc1 / cc1plus (C/C++ Compiler)
  → as (Assembler)
  → ld (Linker) + Libraries → Output ELF File
```

## Compiler Names

| Processor | GCC Compiler |
|-----------|-------------|
| MicroBlaze / MicroBlaze-V | `mb-gcc` |
| Cortex-A9 | `arm-none-eabi-gcc` |
| Cortex-A53 / Cortex-A72 | `aarch64-none-elf-gcc` |
| Cortex-R5F | `armr5-none-eabi-gcc` |

## Pipeline Stages

- **Pre-processor (`cpp0`):** First pass invoked by the compiler. Replaces all macros with definitions as defined in the source and header files.

- **Machine and language-specific compiler:** Works on the pre-processed code (output of the first stage):
  - **C Compiler (`cc1`):** Responsible for most of the optimizations done on the input C code and for generating assembly code.
  - **C++ Compiler (`cc1plus`):** Responsible for most of the optimizations done on the input C++ code and for generating assembly code.

- **Assembler:** Converts mnemonics to machine language, resolves some of the labels generated by the compiler, and creates an object file that passes on to the linker.

  | Processor | Assembler |
  |-----------|-----------|
  | MicroBlaze / MicroBlaze-V | `mb-as` |
  | Cortex-A9 | `arm-none-eabi-as` |
  | Cortex-A53 / Cortex-A72 | `aarch64-none-elf-as` |
  | Cortex-R5F | `armr5-none-eabi-as` |

- **Linker:** Links all object files generated by the assembler. If libraries are provided on the command line, the linker can resolve undefined references by linking in functions from the libraries.

  | Processor | Linker |
  |-----------|--------|
  | MicroBlaze / MicroBlaze-V | `mb-ld` |
  | Cortex-A9 | `arm-none-eabi-ld` |
  | Cortex-A53 / Cortex-A72 | `aarch64-none-elf-ld` |
  | Cortex-R5F | `armr5-none-eabi-ld` |
