# Chapter 18 — HLS Tcl Commands

> Source: *Vitis HLS User Guide* UG1399 v2025.2, Chapter 18 (pp. 608–712)

Chapter 18 covers all Vitis HLS Tcl commands organized in three sections:

1. [Project Commands](#project-commands) — Create and manage projects, solutions, simulation, synthesis, and export.
2. [Configuration Commands](#configuration-commands) — Configure synthesis, interface, simulation, RTL, and storage defaults (`config_*`).
3. [Optimization Directives](#optimization-directives) — `set_directive_*` — TCL equivalents of HLS pragmas.

All commands run in interactive mode (`vitis_hls -i`) or in scripts (`vitis_hls -f`).

---

## Project Commands

### add_files

Adds design source files or test bench files to the current project.

```tcl
add_files [OPTIONS] <src_files>
```

| Option | Description |
|---|---|
| `-blackbox <file.json>` | JSON file for RTL blackbox; used during synthesis and co-sim |
| `-cflags <string>` | Compiler options for synthesis |
| `-csimflags <string>` | Compiler options for simulation only (does not affect synthesis) |
| `-tb` | Mark files as test bench (not synthesized) |

**Examples:**
```tcl
add_files a.cpp b.cpp
add_files -tb "a_test.cpp input.dat out.gold.dat"
add_files -tb test_data/       # adds whole directory as TB data
```

---

### apply_ini

Applies a previously saved INI file to configure the current project+solution.

```tcl
apply_ini <ini_file> [OPTIONS]
# Options: -exclude <pattern>, -include <pattern>, -quiet <T|F>, -show <T|F>
```

> Requires `open_project` and `open_solution` to be called first.

---

### cat_ini

Concatenates multiple INI files into one (output to stdout or file).

```tcl
cat_ini {file1.ini file2.ini} [-out output.ini] [-exclude <pattern>] [-include <pattern>]
```

> `-exclude` and `-include` are mutually exclusive.

---

### close_project / close_solution

Closes the active project or solution. Not strictly necessary (opening another project/solution closes the current one automatically), but saves results immediately.

```tcl
close_project
close_solution
```

### cosim_design

Runs post-synthesis C/RTL co-simulation.

```tcl
cosim_design [OPTIONS]
```

| Option | Description |
|---|---|
| `-O` | Optimized compilation (slower compile, faster runtime) |
| `-argv <string>` | Arguments passed to test bench `main()` |
| `-rtl [verilog\|vhdl]` | HDL language for co-sim (default: `verilog`) |
| `-setup` | Generate sim files only; do not run simulation |
| `-trace_level [none\|all\|port\|port_hier]` | Waveform save level (default: `none`) |
| `-wave_debug` | Open Vivado simulator GUI for waveform viewing (xsim only) |
| `-disable_deadlock_detection` | Disable deadlock detection |
| `-enable_dataflow_profiling` | Track dataflow channel sizes during co-sim |
| `-enable_fifo_sizing` | Automatic FIFO depth tuning during co-sim |
| `-random_stall` | Enable random stalling of top-level interfaces |
| `-user_stall <json>` | Specify JSON stall file (generated by `cosim_stall`) |
| `-hwemu_trace_dir <kernel/instance>` | Use hardware emulation test vectors |
| `-coverage` | Enable coverage (VCS only) |
| `-ldflags <string>` | Linker flags for test bench |
| `-stable_axilite_update` | Allow updating stable s_axilite registers across transactions |

### cosim_stall

Manages stall file generation for co-simulation.

```tcl
cosim_stall -generate <file.json>    # generate a stall file
cosim_stall -check <file.json>       # validate and use during cosim
cosim_stall -list                    # list ports that can be stalled
cosim_stall -generate_burst_default  # more accurate burst traffic modeling
```

### create_clock

Creates a virtual clock constraint for synthesis.

```tcl
create_clock -period <number> [-name <string>]
```

- Period specified in ns (default) or MHz.
- Default period is 10 ns.
- Only a single clock is supported for C/C++ designs.

```tcl
create_clock -period 5         # 5 ns = 200 MHz
create_clock -period 100MHz    # 100 MHz
create_clock                   # default 10 ns
```



### open_project

Opens an existing project or creates a new one.

```tcl
open_project <name>               # open or create
open_project -reset <name>        # open and reset (clear existing data)
open_project -upgrade <name>      # upgrade from Vivado HLS format
open_project -ide [classic|vitis] <name>
```

> **Recommended:** always use `-reset` when running Tcl scripts to avoid accumulating stale `add_files` entries.

---

### open_solution

Opens or creates a solution in the active project.

```tcl
open_solution <name>
open_solution -flow_target vitis   Solution_1    # Vitis kernel flow (.xo export)
open_solution -flow_target vivado  Solution_2    # Vivado IP flow (default)
open_solution -reset               Solution_3    # clear existing data
```

- `vivado` flow: strict pragma usage; output as Vivado IP.
- `vitis` flow: auto-infers interfaces; output as `.xo` kernel object.

## Configuration Commands

Configuration commands (`config_*`) control synthesis behavior, interface defaults, simulation settings, and RTL output properties. They can also be set in the Vitis HLS IDE under **Solution Settings**.

---

### config_array_partition

Controls automatic array partitioning behavior.

```tcl
config_array_partition [OPTIONS]
```

| Option | Default | Description |
|---|---|---|
| `-throughput_driven [auto\|off]` | `auto` | `auto` = smart partitioning; `off` = disable |
| `-complete_threshold <uint>` | `4` | Arrays with fewer elements than this threshold are fully partitioned |

```tcl
config_array_partition -complete_threshold 12    # fully partition arrays < 12 elements
config_array_partition -throughput_driven off    # disable auto partitioning
```

---

### config_compile

Configures front-end compilation and auto-pipelining behavior.

```tcl
config_compile [OPTIONS]
```

| Option | Default | Description |
|---|---|---|
| `-pipeline_loops <threshold>` | `64` | Auto-pipeline loops with trip count > threshold |
| `-pipeline_style [stp\|flp\|frp]` | `stp` | Default pipeline style for auto/explicit pipelining |
| `-pipeline_flush_in_task [always\|never\|ii1]` | `ii1` | Flush pipelines in `hls::task` — `ii1` = only when II=1 |
| `-performance_budgeter [enable\|disable\|autodisable]` | `autodisable` | Design-wide performance analysis when top-level PERFORMANCE pragma found |
| `-float_acc_bits=M` | `45` | Accumulator precision for `float` |
| `-double_acc_bits=N` | `269` | Accumulator precision for `double` |
| `-no_signed_zeros` | `false` | Ignore signed zero (aggressive FP optimization — may cause co-sim mismatch) |
| `-unsafe_math_optimizations` | `false` | Also enable associative FP operations (may cause mismatch) |
| `-name_max_length <value>` | `256` | Max function name length before truncation |
| `-pragma_strict_mode` | `false` | Error on misplaced/misused pragmas |
| `-pre_tcl <script>` | — | Tcl script to run before `csynth_design` |
| `-enable_auto_rewind` | `true` | Use alternative pipelined loop implementation supporting auto-rewind |
| `-use_csim_directives` | `false` | Apply directives generated by C simulation during synthesis |
| `-ignore_long_run_time` | `false` | Suppress long-runtime warning |

### config_dataflow

Controls DATAFLOW channel types and FIFO depths.

```tcl
config_dataflow [OPTIONS]
```

| Option | Default | Description |
|---|---|---|
| `-default_channel [fifo\|pingpong]` | `pingpong` | Default channel type between dataflow tasks |
| `-fifo_depth <int>` | `2` | Default FIFO depth (also default for scalar propagation FIFOs) |
| `-scalar_fifo_depth <int>` | = `fifo_depth` | Min depth for scalar propagation FIFOs |
| `-start_fifo_depth <int>` | = `fifo_depth` | Min depth for start propagation FIFOs |
| `-task_level_fifo_depth <int>` | — | Depth for task-level FIFOs (synchronized by `ap_ctrl_chain`) |
| `-override_user_fifo_depth <int>` | — | Override **all** `hls::stream` depths (useful for deadlock diagnosis) |
| `-strict_mode [off\|warning\|error]` | — | Severity for dataflow canonical form violations |
| `-disable_fifo_sizing_opt` | — | Disable FIFO size optimizations |
| `-enable_canonicalization` | `true` | Enable canonical form transformations |
| `-strict_stable_sync` | `false` | Force stable port synchronization with `ap_done` |

**Deadlock diagnosis:**
```tcl
config_dataflow -fifo_depth 1000 -start_fifo_depth 1000 -scalar_fifo_depth 1000 -task_level_fifo_depth 1000
# if deadlock disappears, it was caused by insufficient depth
```

### config_op

Sets global default operation-to-implementation bindings.

```tcl
config_op <op> [-impl <value>] [-latency <value>] [-precision [low|high|standard]]
```

Supported `<op>` values: `mul`, `add`, `sub`, `fadd`, `fsub`, `fdiv`, `fexp`, `flog`, `fmul`, `frsqrt`, `frecip`, `fsqrt`, `dadd`, `dsub`, `ddiv`, `dexp`, `dlog`, `dmul`, `drsqrt`, `drecip`, `dsqrt`, `hadd`, `hsub`, `hdiv`, `hmul`, `hsqrt`, `facc`, `fmacc`, `fmadd`

`-impl` values: `dsp`, `fabric`, `meddsp`, `fulldsp`, `maxdsp`, `primitivedsp`, `auto`, `none`, `all`

- `auto` / `none` — enable/disable inference of combined `facc`/`fmacc`/`fmadd` operators.
- `-precision [low|high|standard]` — applies to `facc`, `fmacc`, `fmadd`:
  - `low`: 60/100-bit integer accumulation, II=1 on non-Versal (uses ~3× resources, may mismatch co-sim).
  - `high`: extra-precision fused multiply-add; may mismatch co-sim.
  - `standard` (default): true floating-point; II=1 on Versal; II=3–5 on non-Versal.
- DCM comparison operators (`dcmp` etc.) are LUT-only and not configurable.

```tcl
config_op add -impl fabric -latency 2
config_op facc -impl auto -precision low    # II=1 on non-Versal
```

## Optimization Directives

`set_directive_*` commands are the Tcl equivalents of HLS pragmas. They are stored in `solution/constraints/script.tcl` and allow different optimizations for different solutions without modifying source code.

Syntax convention: `set_directive_<pragma> [OPTIONS] <location> [<variable>]`

where `<location>` format is `function[/label]`.

---

### set_directive_aggregate

```tcl
set_directive_aggregate [OPTIONS] <location> <variable>
# -compact [bit|byte|none|auto]
```
→ Equivalent to `#pragma HLS aggregate`

---

### set_directive_alias

```tcl
set_directive_alias [OPTIONS] <location> <ports>
# -distance <int>  OR  -offset <string>
```
→ Equivalent to `#pragma HLS alias`

---

### set_directive_allocation

```tcl
set_directive_allocation [-limit <int>] [-type [function|operation]] <location> <instances>
```
→ Equivalent to `#pragma HLS allocation`

---

### set_directive_array_partition

```tcl
set_directive_array_partition [OPTIONS] <location> <array>
# -type [block|cyclic|complete]  -factor <int>  -dim <int>  -off
```
→ Equivalent to `#pragma HLS array_partition`

### set_directive_array_reshape

```tcl
set_directive_array_reshape [OPTIONS] <location> <array>
```

### set_directive_bind_storage

```tcl
set_directive_bind_storage [-type <type>] [-impl <value>] [-latency <int>] <location> <variable>
```

**Supported implementations by scope:**

| Type | Scope | Supported Impls |
|---|---|---|
| FIFO | local (`bind_storage`) | AUTO, BRAM, LUTRAM, URAM, MEMORY, SRL |
| FIFO | global (`config_storage`) | AUTO, BRAM, LUTRAM, URAM, MEMORY, SRL |
| RAM*/ROM* | local (`bind_storage`) | AUTO, BRAM, BRAM_ECC, LUTRAM, URAM, URAM_ECC |
| s_axilite storage | local (`interface -storage_impl`) | AUTO, BRAM, URAM |
| M_AXI buffer | global (`config_interface -m_axi_buffer_impl`) | AUTO, BRAM, LUTRAM, URAM |

> **URAM note:** No read-first write mode. If read and write to same address map to same port, use BRAM instead (HLS warns when scheduling conflict occurs on URAM port).

## Best Practices

1. **Always use `-reset` with `open_project`** in Tcl scripts — prevents accumulation of stale `add_files` from previous runs.
2. **`config_dataflow` FIFO depth** — default depth of 2 is often insufficient for burst-heavy designs; increase conservatively and use co-sim profiling (`-enable_dataflow_profiling`) to find minimum.
3. **Deadlock diagnosis with FIFO depth override** — set all FIFO depths to a large value with `config_dataflow -override_user_fifo_depth <large_N>`; if deadlock disappears, it is depth-related.
4. **Conservative M_AXI mode** — `config_interface -m_axi_conservative_mode true` (default) avoids write deadlocks but adds latency. Disable only if explicit buffering >= max write burst length is already in place.
5. **64-bit M_AXI addressing** — enable globally with `config_interface -m_axi_addr64` for Vitis flows accessing >4 GB address spaces.
6. **Directives vs. Pragmas** — Use `set_directive_*` for what-if exploration across multiple solutions; use `#pragma HLS` for permanent optimization decisions embedded in source code.
7. **`config_rtl -reset_level low`** — Required for AXI compliant interfaces. HLS warns if using active-High reset with AXI interfaces.
